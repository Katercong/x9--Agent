## 💡 通用约束（任何会话开始都遵守）

下面这段建议你在每次新对话开头作为系统提示带上：

```
项目硬约束：
1. 服务器路径 \\192.168.1.171\FShare\Database\，端口 18765。
2. 我的身份是 zhang (admin)，token 在 .local_keys_backup.txt。
3. 数据库是 SQLite (database.db)，schema 通过 migrate_v*.py 增量演进，不直接改 schema.sql。
4. 写代码遵守：API 是契约，不破坏既有 endpoint；加字段/表/查询是非破坏性的随便加；删/改字段名要先标 deprecated_note。
5. .bat 文件必须 CRLF 换行 + ASCII + 用 >NUL 不要 >/dev/null。
6. 不要给业务决策（要不要给某人 admin / 这个达人能不能合作） — 这是我的判断。
7. 尽量复用 generic CRUD (/api/v1/data/{r}) 而不是写新 endpoint。
8. 加 AI 功能走 llm_feature 绑定 + 复用 _call() helper，不要直接调 LLM SDK。
```

## 🎯 一句话原则

**不要让新会话去消化历史对话。让旧对话的成果（这套交接文档）按需供给新会话。**

每次新对话最理想的状态：
- 第一条消息：发上方某段 kickoff
- 第二条消息：AI 读完文档说"已对齐"
- 第三条消息：你给具体任务
- AI 直接干活，不绕圈
