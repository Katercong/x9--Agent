## 一、基础信息

### 1.1 服务地址

| 环境 | 地址 |
|---|---|
| 生产环境 | `http://192.168.1.168:18765` |
| 本地测试 | `http://localhost:18765` |

### 1.2 认证方式

所有写操作需要在 HTTP Header 中携带 API Key：

```
X-API-Key: <你的 API Key>
Content-Type: application/json
```

**获取 API Key**：
- 联系张，在「设置 → 用户管理 → 廖 → 签发新 Key」生成
- Key 只显示一次，请妥善保存
- Key 泄露时立即联系张撤销并重新签发

### 1.3 通用响应格式

**成功**：
```json
{
  "upserted": 3,
  "updated": 2,
  "inserted": 1
}
```

**失败**：
```json
{
  "detail": "错误描述"
}
```

### 1.4 错误码

| 状态码 | 含义 | 处理建议 |
|---|---|---|
| 200 | 成功 | — |
| 401 | API Key 无效或未传 | 检查 Header 中的 X-API-Key |
| 403 | 权限不足 | 联系张确认角色权限 |
| 404 | 资源不存在 | 检查 URL 路径 |
| 422 | 请求体格式错误 | 检查 JSON 格式和字段名 |
| 500 | 服务器内部错误 | 联系张排查 |

## 二、达人数据写入

### 2.1 端点

```
POST /api/v1/data/creators/bulk
```

### 2.2 用途

将爬虫抓取的达人基础信息批量写入 `creator` 表。

### 2.3 请求体

```json
[
  {
    "handle": "la_patrona_diana",
    "platform": "tiktok",
    "profile_url": "https://www.tiktok.com/@la_patrona_diana",
    "display_name": "Diana",
    "country": "US",
    "language": "en",
    "followers": 125000,
    "tier": "A",
    "engagement_rate": 0.045,
    "avg_views": 50000,
    "email": "diana@example.com",
    "whatsapp": "+1-555-123-4567",
    "instagram_handle": "@diana_ig",
    "category_tags": "[\"pet\", \"lifestyle\"]",
    "source": "crawler_2026-05-08",
    "quality_score": 0.85
  }
]
```

### 2.4 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `handle` | string | ✅ | 达人唯一标识（TikTok @后面的部分），**Upsert Key** |
| `platform` | string | ✅ | 平台：`tiktok` / `instagram` / `youtube` |
| `profile_url` | string | ❌ | 主页链接 |
| `display_name` | string | ❌ | 显示名称 |
| `country` | string | ❌ | 国家代码：`US` / `UK` / `DE` / `ES` 等（**不爬，由张这边根据其他字段推断**） |
| `language` | string | ❌ | 语言代码：**固定 `en`，不爬** |
| `followers` | integer | ❌ | 粉丝数 |
| `tier` | string | ❌ | 达人等级：`A` / `B` / `C`（可由张这边根据 followers 自动计算） |
| `engagement_rate` | float | ❌ | 互动率（0~1），如 0.045 = 4.5% |
| `avg_views` | integer | ❌ | 平均播放量 |
| `email` | string | ❌ | 联系邮箱 |
| `whatsapp` | string | ❌ | WhatsApp 号码 |
| `instagram_handle` | string | ❌ | Instagram 账号 |
| `category_tags` | string | ❌ | 内容标签，JSON 数组格式：`"[\"pet\", \"lifestyle\"]"` |
| `source` | string | ❌ | 数据来源标识，建议格式：`crawler_YYYY-MM-DD` |
| `quality_score` | float | ❌ | 质量评分（0~1） |

### 2.5 Upsert 规则

- **Key 字段**：`handle`
- `handle` 已存在 → 更新该行所有传入字段
- `handle` 不存在 → 新建一条记录

### 2.6 调用示例

```python
import requests

BASE_URL = "http://192.168.1.168:18765"
API_KEY = "你的API Key"

creators = [
    {
        "handle": "test_creator_001",
        "platform": "tiktok",
        "followers": 50000,
        "country": "US",
        "language": "en"
    }
]

resp = requests.post(
    f"{BASE_URL}/api/v1/data/creators/bulk",
    headers={"X-API-Key": API_KEY, "Content-Type": "application/json"},
    json=creators
)
print(resp.json())
```

## 三、视频曝光数据写入

### 3.1 端点

```
POST /api/v1/data/outreach/bulk
```

### 3.2 用途

将爬虫抓取的视频表现数据写入 `outreach` 表，用于：
- 追踪已合作达人的视频表现
- 触发自动话术（感谢 / 再合作邀约）

### 3.3 请求体

```json
[
  {
    "creator_id": 5,
    "video_url": "https://www.tiktok.com/@la_patrona_diana/video/738291100",
    "action": "video_track",
    "channel": "tiktok",
    "event_date": "2026-05-08",
    "video_views": 45230,
    "video_likes": 3210,
    "video_comments": 445,
    "video_shares": 890,
    "gmv_30d_usd": 1280.50,
    "metrics_updated_at": "2026-05-08 10:00:00"
  }
]
```

### 3.4 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `creator_id` | integer | ✅ | 达人 ID（需先通过 `creators` 表获取或创建） |
| `video_url` | string | ✅ | 视频链接，**Upsert Key** |
| `action` | string | ❌ | 固定值 `video_track`，标识这是视频追踪记录 |
| `channel` | string | ❌ | 平台：`tiktok` / `instagram` / `youtube` |
| `event_date` | string | ❌ | 视频发布日期：`YYYY-MM-DD` |
| `video_views` | integer | ❌ | 播放量 |
| `video_likes` | integer | ❌ | 点赞数 |
| `video_comments` | integer | ❌ | 评论数 |
| `video_shares` | integer | ❌ | 分享数 |
| `gmv_30d_usd` | float | ❌ | 30 天 GMV（美元） |
| `metrics_updated_at` | string | ❌ | 指标更新时间：`YYYY-MM-DD HH:MM:SS` |

### 3.5 Upsert 规则

- **Key 字段**：`video_url`
- `video_url` 已存在 → 更新该行的指标字段
- `video_url` 不存在 → 新建一条 outreach 记录

### 3.6 调用示例

```python
import requests
from datetime import datetime

BASE_URL = "http://192.168.1.168:18765"
API_KEY = "你的API Key"

now = datetime.now().strftime("%Y-%m-%d %H:%M:%S")

videos = [
    {
        "creator_id": 5,
        "video_url": "https://www.tiktok.com/@test/video/123456",
        "action": "video_track",
        "channel": "tiktok",
        "event_date": now[:10],
        "video_views": 10000,
        "video_likes": 500,
        "video_comments": 50,
        "video_shares": 20,
        "gmv_30d_usd": 150.0,
        "metrics_updated_at": now
    }
]

resp = requests.post(
    f"{BASE_URL}/api/v1/data/outreach/bulk",
    headers={"X-API-Key": API_KEY, "Content-Type": "application/json"},
    json=videos
)
print(resp.json())
```

## 四、热搜关键词写入

### 4.1 端点

```
POST /api/v1/data/tk_hot_keywords/bulk
```

### 4.2 用途

将 TikTok 热搜关键词数据写入 `tk_hot_keyword` 表，用于：
- 标题优化（埋入热搜词）
- 趋势分析

### 4.3 请求体

```json
[
  {
    "keyword": "period underwear",
    "source_platform": "tiktok",
    "region": "US",
    "search_volume": 125000,
    "growth_rate": 0.23,
    "rank": 15,
    "category_hint": "female_care",
    "captured_at": "2026-05-08 10:00:00"
  }
]
```

### 4.4 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `keyword` | string | ✅ | 关键词文本 |
| `source_platform` | string | ✅ | 来源平台：`tiktok` / `amazon` / `google` |
| `region` | string | ✅ | 地区：`US` / `UK` / `DE` / `ES` 等 |
| `search_volume` | integer | ❌ | 搜索量 |
| `growth_rate` | float | ❌ | 增长率（如 0.23 = 23% 增长） |
| `rank` | integer | ❌ | 排名 |
| `category_hint` | string | ❌ | 品类提示：`female_care` / `pet` / `baby` / `adult_care` / `home_care` / `mask` / `general` |
| `captured_at` | string | ❌ | 抓取时间：`YYYY-MM-DD HH:MM:SS` |

### 4.5 Upsert 规则

- **Key 字段组合**：`(keyword, source_platform, region)`
- 组合已存在 → 更新 search_volume / growth_rate / rank
- 组合不存在 → 新建一条

### 4.6 调用示例

```python
import requests
from datetime import datetime

BASE_URL = "http://192.168.1.168:18765"
API_KEY = "你的API Key"

now = datetime.now().strftime("%Y-%m-%d %H:%M:%S")

keywords = [
    {
        "keyword": "pet diapers",
        "source_platform": "tiktok",
        "region": "US",
        "search_volume": 50000,
        "growth_rate": 0.15,
        "rank": 25,
        "category_hint": "pet",
        "captured_at": now
    }
]

resp = requests.post(
    f"{BASE_URL}/api/v1/data/tk_hot_keywords/bulk",
    headers={"X-API-Key": API_KEY, "Content-Type": "application/json"},
    json=keywords
)
print(resp.json())
```

## 六、完整调用模板

以下是一个完整的 Python 模块，可以直接复制使用：

```python
"""
X9 数据库爬虫写入模块
使用方法：将本文件保存为 x9_client.py，在其他脚本中 import 使用
"""

import requests
from datetime import datetime
from typing import List, Dict, Optional

class X9Client:
    def __init__(self, base_url: str = "http://192.168.1.168:18765", api_key: str = ""):
        self.base_url = base_url
        self.api_key = api_key
        self.headers = {
            "X-API-Key": api_key,
            "Content-Type": "application/json"
        }
    
    def _post(self, endpoint: str, data: List[Dict]) -> Dict:
        """通用 POST 请求"""
        url = f"{self.base_url}{endpoint}"
        resp = requests.post(url, headers=self.headers, json=data)
        return resp.json()
    
    def push_creators(self, creators: List[Dict]) -> Dict:
        """
        写入达人数据
        
        参数:
            creators: 达人列表，每个达人需包含:
                - handle (必填): 达人唯一标识
                - platform (必填): 平台
                - followers, country, language, email, whatsapp 等可选
        
        返回:
            {"upserted": N, "updated": N, "inserted": N}
        """
        return self._post("/api/v1/data/creators/bulk", creators)
    
    def push_video_metrics(self, videos: List[Dict]) -> Dict:
        """
        写入视频曝光数据
        
        参数:
            videos: 视频列表，每个视频需包含:
                - creator_id (必填): 达人 ID
                - video_url (必填): 视频链接
                - video_views, video_likes, video_comments, video_shares, gmv_30d_usd 可选
        
        返回:
            {"upserted": N, "updated": N, "inserted": N}
        """
        now = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        payload = []
        for v in videos:
            item = {
                "creator_id": v["creator_id"],
                "video_url": v["video_url"],
                "action": "video_track",
                "channel": v.get("channel", "tiktok"),
                "event_date": v.get("event_date", now[:10]),
                "video_views": v.get("views", 0),
                "video_likes": v.get("likes", 0),
                "video_comments": v.get("comments", 0),
                "video_shares": v.get("shares", 0),
                "gmv_30d_usd": v.get("gmv_30d_usd"),
                "metrics_updated_at": now
            }
            payload.append(item)
        return self._post("/api/v1/data/outreach/bulk", payload)
    
    def push_keywords(self, keywords: List[Dict]) -> Dict:
        """
        写入热搜关键词
        
        参数:
            keywords: 关键词列表，每个需包含:
                - keyword (必填): 关键词文本
                - source_platform (必填): 来源平台
                - region (必填): 地区
                - search_volume, growth_rate, rank, category_hint 可选
        
        返回:
            {"upserted": N, "updated": N, "inserted": N}
        """
        now = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        payload = []
        for k in keywords:
            item = {
                "keyword": k["keyword"],
                "source_platform": k["source_platform"],
                "region": k["region"],
                "search_volume": k.get("search_volume"),
                "growth_rate": k.get("growth_rate"),
                "rank": k.get("rank"),
                "category_hint": k.get("category_hint"),
                "captured_at": now
            }
            payload.append(item)
        return self._post("/api/v1/data/tk_hot_keywords/bulk", payload)
    
    def push_competitor_collabs(self, collabs: List[Dict]) -> Dict:
        """
        写入竞品合作记录
        
        参数:
            collabs: 合作记录列表，每个需包含:
                - creator_id (必填): 达人 ID
                - competitor_brand_id (必填): 竞品品牌 ID
                - evidence_url, confidence, notes 可选
        
        返回:
            {"upserted": N, "updated": N, "inserted": N}
        """
        return self._post("/api/v1/data/creator_competitor_collab/bulk", collabs)


# ========== 使用示例 ==========

if __name__ == "__main__":
    # 初始化客户端
    client = X9Client(
        base_url="http://192.168.1.168:18765",
        api_key="你的API Key"
    )
    
    # 示例 1：写入达人
    creators = [
        {
            "handle": "test_creator_001",
            "platform": "tiktok",
            "followers": 50000,
            "country": "US",
            "language": "en",
            "email": "test@example.com"
        }
    ]
    result = client.push_creators(creators)
    print(f"达人写入结果: {result}")
    
    # 示例 2：写入视频指标
    videos = [
        {
            "creator_id": 1,
            "video_url": "https://www.tiktok.com/@test/video/123456",
            "views": 10000,
            "likes": 500,
            "comments": 50,
            "shares": 20,
            "gmv_30d_usd": 150.0
        }
    ]
    result = client.push_video_metrics(videos)
    print(f"视频指标写入结果: {result}")
    
    # 示例 3：写入热搜关键词
    keywords = [
        {
            "keyword": "pet diapers",
            "source_platform": "tiktok",
            "region": "US",
            "search_volume": 50000,
            "growth_rate": 0.15,
            "category_hint": "pet"
        }
    ]
    result = client.push_keywords(keywords)
    print(f"关键词写入结果: {result}")
```

## 八、常见问题

### Q1: 如何获取 creator_id？

**方法 A**：写入达人时，X9 会自动分配 ID。你可以通过查询获取：

```python
# 查询达人 ID
resp = requests.get(
    "http://192.168.1.168:18765/api/v1/data/creators",
    headers={"X-API-Key": API_KEY},
    params={"q": "la_patrona_diana"}  # 按 handle 搜索
)
creators = resp.json()["items"]
creator_id = creators[0]["id"]
```

**方法 B**：如果爬虫需要保持 ID 映射，建议在本地维护一份 `handle → id` 的缓存表。

### Q2: 重复写入会怎样？

所有 bulk 端点都是 **Upsert** 语义：
- Key 字段已存在 → 更新
- Key 字段不存在 → 插入

所以可以放心重复写入，不会产生重复数据。

### Q3: 写入失败怎么办？

建议在爬虫脚本中加入重试逻辑：

```python
import time

def write_with_retry(client, data, max_retries=3):
    for i in range(max_retries):
        try:
            result = client.push_creators(data)
            return result
        except Exception as e:
            print(f"写入失败 (第 {i+1} 次): {e}")
            time.sleep(5 * (i + 1))  # 指数退避
    raise Exception("写入失败，已达最大重试次数")
```
