## 3. 实时连通性测试(Phase 2)

工具:`infra/scripts/test_api_local.ps1`(auth 感知:以 `superadmin` 登录复用 `x9_session` Cookie;读接口实打,写接口走"建后即删/认领即释放"或"假 ID/非法体探活",对 restart/send/migrate/run-full-pipeline/disconnect/logout 设硬护栏只记录不调用)。

**结果(可复现,连跑 3 次一致):TOTAL 106 | PASS 83 | FAIL 4 | WARN 8 | SKIPPED 11。**

- 代理链路正常:`/api/v1/*` 全 200(~450–540ms,慢在多一跳到 Core)。

- SPA 路由正常:`/portal/dashboard`、`/`、`/portal/` 均 200;`/workspace/cross-border/` → 307→`/portal/dashboard`。

### 3.1 四个 HTTP 500(P1,线上真实报错,均已定位根因)

| 端点 | 根因 | 位置 |
|---|---|---|
| `GET /api/local/review-tasks` | `created_at/reviewed_at` 实为 `str`(SQLite TEXT),代码却调 `.isoformat()` → `AttributeError` | `review_tasks.py:37`(经 `:51`) |
| `GET /api/local/admin/trends` | Postgres:`raw_observation.created_at` 列为 `text`,查询做 `text >= timestamp` 无此运算符 → `ProgrammingError` | `admin.py:402` |
| `GET /api/local/extension/commands` | datetime 实为 `str`,`.isoformat()` 失败 | `extension.py:179`(经 `:272`) |
| `GET /api/local/extension/run-progress` | 同上 | `extension.py:312`(经 `:394`) |

3 个属同一类缺陷(序列化函数假设 ORM 返回 `datetime`,实际拿到字符串);trends 是 Postgres 列类型漂移(text vs timestamp)。**这 4 个在当前源码中即真实存在(进程内 TestClient 同样抛异常)**,且都是 portal 直接调用的页面(审核队列、采集趋势、扩展命令、运行进度监控)→ 这些看板模块线上报错。
修复方向:序列化前判断类型/统一用 `loads`/兼容 str;`raw_observation.created_at` 在 PG 改为 `timestamptz` 或查询侧 `CAST(... AS timestamp)`。

### 3.2 `/api/local/data/{resource}` 全 404 —— 实为线上进程陈旧(P1)

实测:进程内(当前磁盘源码)`/api/local/data/{creators,users,review_tasks}` 均 **200**(creators 182 行、users 14 行);但经线上 :8000 HTTP **空体 404**。结论:**运行中的 uvicorn 是用更早代码启动的**(有 `/portal`、`/api/v1` 代理,但没有后加的 `data` 路由),改了源码未重启。影响:依赖 `X9DB_RESOURCES`(`/api/local/data/*`)的 admin SPA 数据页线上全部失效;也说明本报告所有"源码侧"结论需重启进程才反映到线上。

## 5. 问题与风险清单(Phase 4,按优先级)

### P0(立即处理)

**P0-1 默认超管口令硬编码,且线上生效**
- 位置:`desktop/backend/config.py:52-53`、`desktop/backend/services/auth_service.py:143-181`
- 事实:`super_admin_password` 默认 `"X9@2026"`;`.env.shared`/`desktop/.env` **均未设** `X9_SUPER_ADMIN_PASSWORD/USERNAME`;`must_change_password` 在登录路径**不强制**(仅 UI 提示)。已**实测**:对 `http://127.0.0.1:8000`(=https://usx9.us)`POST /api/local/auth/login` `superadmin/X9@2026` → 200,`role=super_admin`,拿到 14 天会话。
- 影响:**公网完全接管**——重置任意用户密码、读写全部部门达人/线索/邮件、推送扩展命令、CSV 导出。口令亦在 git 历史中。
- 修复:删除源码默认值,未设环境变量则启动即失败;立即在线上轮换该口令;登录路径强制 `must_change_password`;`X9@2026` 视为已泄露并清理 git 历史。

**P0-2 CORS 通配 + 携带凭证**
- 位置:`desktop/backend/main.py:42-48`(`allow_origins=["*"]` 且 `allow_credentials=True`)
- 影响:已登录用户访问恶意站点时,该站点 JS `fetch('https://usx9.us/api/local/creators',{credentials:'include'})` 可携带受害者 Cookie 跨域**读取**全部达人/线索/邮件数据。`samesite=lax` 不能阻止跨域读。
- 修复:改为显式白名单(`https://usx9.us` + 开发源),保留 `allow_credentials=True` 仅配合白名单;对 `/api/local/*` 写接口加 CSRF Token 作纵深防御。

**P0-3 未鉴权可变更接口(数据投毒/未授权写)**
- 位置:`main.py:71-89` 白名单 + `collector.py:39`、`extension.py:185/217/242/344/433/501`
- 已实测未登录返回 200/可达:`POST /extension/x9-compat/ingest-creators`(批量注入达人,字段含 `email/profile_url/department_code`)、`POST /collector/observations`(注入原始观测+upsert creator 到 `cross_border`)、`/heartbeat`/`/launcher-heartbeat`/`/run-progress`(伪造在线扩展会话)。前缀 `/api/local/extension/commands/` 整体 PUBLIC → `GET /commands/pending`(会把 pending 改 claimed 并回传 payload)与 `POST /commands/{id}/ack`(篡改任意命令结果)亦免鉴权。
- 影响:公网未授权写核心业务表,污染推荐/外联管线与运营看板,命令队列可被窃取/篡改;无体积限制还可存储 DoS。
- 修复:这些 ingest/命令轮询/ack/心跳改为校验每扩展共享密钥/签名;收窄 `PUBLIC_API_PREFIXES` 使 `commands/` 不再整体放行;加体积与限流。
