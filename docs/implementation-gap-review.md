# X9 ReplyChat Agent 阶段性实现复盘

> 对照：[最终需求规格](final-requirements.md)。本文记录事实状态，不等同于目标需求。代码基线：`main` 的 `29bd7c1`（2026-07-20）。

## 已实现

| 模块 | 当前能力 |
| --- | --- |
| 数据库 | Docker Compose 本地 PostgreSQL、`.env` 配置、Alembic 初始迁移；SQLite 自动化测试兼容。 |
| 数据完整性 | 外部消息 ID 非空与幂等；run 外键非空；审计关联禁止级联删除；每条回复最多一个活跃 run。 |
| 核心数据 | 达人、产品、参考资料版本、入站回复、Agent run、DNC、待办、事件与模拟出站指令。 |
| 模拟流程 | 模拟入站幂等入库、规则分类、上下文构建、队列化 run、Worker、AI 草稿、人工审核状态。 |
| 终态分流 | 退信忽略；拒绝当前直接标记 `dropped`；退订创建待确认 DNC；没有自动外发。 |
| AI | 硅基流动 OpenAI 兼容调用、JSON Mode、Pydantic 校验、提示词脱敏和长度限制、失败留痕。 |
| Worker 可靠性 | 短事务领取、lease、claim token 条件回写、过期回收、非预期异常留痕。 |
| 验证 | 当前基线全量测试为 `64 passed`；仅有 FastAPI `on_event` 既有弃用警告。 |

## 已实现但与目标仍有差距

| 项目 | 当前状态 | 下一步 |
| --- | --- | --- |
| 默认模型与提示词 | 代码默认仍为 V1 + V4 Flash；V2 与 V3.2 已验证。 | 将默认改为 V2 + V3.2、关闭 thinking，并补回归评测。 |
| 人工审核 | 有 API 状态和待办，无工作台。 | 建设队列、草稿编辑、确认、复制/导出和审计。 |
| 终态审核 | 拒绝仍会直接 `dropped`。 | 增加审核人确认拒绝、退信复核和 DNC 解除流程。 |
| 权限隔离 | 有 `department_code`，无用户/成员关系和 RBAC。 | 建模用户、部门成员与角色，并在接口实施隔离。 |
| 多 Worker | 当前可靠性机制适合单 Worker。 | 实现 PostgreSQL 并发领取、容器化部署、指标与告警。 |
| 审计 | 已有业务留痕。 | 增加不可篡改的人工决定、导出和管理员操作审计。 |

## 未实现需求

- 企业身份提供方待定的认证集成、运营/审核/管理员角色和跨部门隔离。
- 外接渠道待选。选定后才可实现适配器、认证、初始回填、增量同步、游标、重试、回补和契约测试。
- 真实消息解析、稳定消息/会话标识保存、待匹配队列及人工关联/建档。
- Web 工作台、站内通知、队列筛选、负责人、优先级和到期日。
- 草稿编辑、确认、复制或结构化导出，以及导出审计；仍不建设自动发送。
- 24 个月保留策略、定期清理、Secrets 管理、备份恢复、监控告警和端到端容量测试。

## 实施约束

外接渠道发送能力不在未实现清单中：最终需求明确禁止系统自动发送。后续功能仅能实现人工确认后的复制或导出交接。
