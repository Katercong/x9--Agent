# X9 ReplyChat Agent 阶段性实现复盘

> 对照：[最终需求规格](final-requirements.md)。本文记录事实状态，不等同于目标需求。本文覆盖功能分支 `feat/human-review-workflow` 截至 `e3d84e9` 的已验证实现（2026-07-21）；该分支尚未合并到 `main`。

## 已实现

| 模块 | 当前能力 |
| --- | --- |
| 数据库 | Docker Compose 本地 PostgreSQL、`.env` 配置、Alembic 初始迁移；SQLite 自动化测试兼容。 |
| 数据完整性 | 外部消息 ID 非空与幂等；run 外键非空；审计关联禁止级联删除；每条回复最多一个活跃 run。 |
| 核心数据 | 达人、产品、参考资料版本、入站回复、Agent run、DNC、待办、事件、人工审核决定、草稿导出记录与历史模拟出站指令。 |
| 模拟流程 | 模拟入站幂等入库、规则分类、上下文构建、队列化 run、Worker、AI 草稿、人工审核队列、人工决定与草稿导出快照。 |
| 人工审核后端 | 普通回复可查看待审队列、批准最终草稿或关闭不使用草稿；决定按 run 唯一留痕，批准后的复制/导出会保存内容快照，系统不会发送消息。 |
| 终态分流 | 退信忽略；拒绝和明确退订均进入只读终态待审队列，不再自动标记 `dropped`、自动确认 DNC 或创建出站指令；待确认/已确认 DNC 会阻断新 run、导出和后续入站的 AI 处理。 |
| AI | 硅基流动 OpenAI 兼容调用、JSON Mode、Pydantic 校验、提示词脱敏和长度限制、失败留痕。 |
| Worker 可靠性 | 短事务领取、lease、claim token 条件回写、过期回收、非预期异常留痕。 |
| 验证 | 全量 SQLite 测试为 `68 passed`；空 PostgreSQL 数据卷已重新 `alembic upgrade head` 到 `9c7a4d1e2b3f`，并验证消息、run 和人工审核审计约束；仅有 FastAPI `on_event` 既有弃用警告。 |

## 已实现但与目标仍有差距

| 项目 | 当前状态 | 下一步 |
| --- | --- | --- |
| 默认模型与提示词 | 代码默认仍为 V1 + V4 Flash；V2 与 V3.2 已验证。 | 将默认改为 V2 + V3.2、关闭 thinking，并补回归评测。 |
| 人工审核 | 已有后端待审队列、普通回复决议、最终草稿与导出快照审计；无前端工作台，`actor_id` 只是审计字段而非认证身份。 | 建设工作台、草稿编辑体验、认证与 RBAC，并将授权主体接入接口。 |
| 终态审核 | 拒绝/DNC 已改为只读待审并阻断后续自动处理；尚无审核人确认拒绝、DNC 确认/解除和退信复核写接口。 | 在 RBAC 落地后设计终态确认状态机、写接口与审计。 |
| 权限隔离 | 有 `department_code`，无用户/成员关系和 RBAC。 | 建模用户、部门成员与角色，并在接口实施隔离。 |
| 多 Worker | 当前可靠性机制适合单 Worker。 | 实现 PostgreSQL 并发领取、容器化部署、指标与告警。 |
| 审计 | 已有业务留痕、按 run 唯一的人工决定和草稿导出内容快照；数据库外键默认 RESTRICT/NO ACTION 保护关联记录。 | 增加管理员操作审计、权限上下文和保留/清理策略。 |

## 未实现需求

- 企业身份提供方待定的认证集成、运营/审核/管理员角色和跨部门隔离。
- 外接渠道待选。选定后才可实现适配器、认证、初始回填、增量同步、游标、重试、回补和契约测试。
- 真实消息解析、稳定消息/会话标识保存、待匹配队列及人工关联/建档。
- Web 工作台、站内通知、队列筛选、负责人、优先级和到期日。
- 认证、部门成员关系、RBAC，以及审核人确认拒绝、DNC 确认/解除和退信复核；当前终态队列只读。
- 草稿编辑的前端体验和受 RBAC 保护的导出交接；已有后端最终草稿确认和导出快照审计，仍不建设自动发送。
- 24 个月保留策略、定期清理、Secrets 管理、备份恢复、监控告警和端到端容量测试。

## 实施约束

外接渠道发送能力不在未实现清单中：最终需求明确禁止系统自动发送。后续功能仅能实现人工确认后的复制或导出交接。
