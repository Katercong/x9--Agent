# 03 — 本地开发环境

> 全员必读。新人入职第一天就跟着这份走。
> 关键约束:Windows + Docker Desktop + Python 3.11 + Node 20

---

## 0. 这个项目长什么样(给新人扫盲)

```
core/        ← FastAPI 后端(端口 18765),业务主数据 + AI + LLM 路由
desktop/     ← FastAPI 后端(端口 8000),桌面/扩展接入 + Gmail 外联 + 评分
web/         ← Vite + React,管理后台前端
web-user/    ← Vite + React,用户/达人门户前端
scrapers/    ← Python 命令行抓取工具(YouTube / TikTok)
infra/       ← Docker compose、数据库初始化、部署脚本
docs/        ← 本套文档以及架构/API/迁移说明
```

**共享一个 PostgreSQL 16 数据库**(Docker 容器 `x9-postgres`),所有服务通过 `.env.shared` 拿连接信息。

## 1. 准备工作(只做一次)

### 1.1 必装软件

| 软件 | 版本 | 下载 |
|---|---|---|
| Git for Windows | 最新 | https://git-scm.com/download/win |
| Docker Desktop | 4.x | https://www.docker.com/products/docker-desktop/ |
| Python | **3.11.x**(不要用 3.12+) | https://www.python.org/downloads/ |
| Node.js | **20.x LTS** | https://nodejs.org/en/download/ |
| VS Code | 最新 | https://code.visualstudio.com/ |
| Windows Terminal | 最新 | Microsoft Store |

**装完后开新的 PowerShell,验证版本:**

```powershell
git --version          # ≥ 2.40
docker --version       # ≥ 24
python --version       # 3.11.x
node --version         # v20.x
npm --version          # ≥ 10
```

如果有不对的,**装对版本再继续**。版本不一致是 90% 跑不起来的原因。

### 1.2 推荐的 VS Code 扩展

- Python(Microsoft)
- Pylance
- Ruff
- ESLint
- Prettier
- GitLens
- Docker
- PowerShell

### 1.3 配置 Git 用户身份

```powershell
git config --global user.name "你的姓名"
git config --global user.email "你的工作邮箱"
git config --global pull.rebase false        # 默认 merge 拉取(对新人更友好)
git config --global init.defaultBranch main
git config --global core.autocrlf input      # Windows 上避免 CRLF 问题
```

GitHub 账号需要绑定**同一个邮箱**(commit author 才会被识别)。

### 1.4 配置 GitHub SSH(推荐,免每次输密码)

```powershell
ssh-keygen -t ed25519 -C "你的工作邮箱"
# 一路回车,产生 ~/.ssh/id_ed25519 和 id_ed25519.pub

cat ~/.ssh/id_ed25519.pub | Set-Clipboard
# 把内容粘到 https://github.com/settings/keys → New SSH key
```

测试:

```powershell
ssh -T git@github.com
# 应该看到 "Hi <你的用户名>! You've successfully authenticated..."
```

## 2. 克隆和首次启动

### 2.1 克隆仓库

```powershell
# 新建工作目录(团队成员推荐放 D:\code\,Rocky 见第 6 节)
mkdir D:\code -ErrorAction SilentlyContinue
cd D:\code

# 用 SSH 克隆(假设你做完 1.4)
git clone git@github.com:X9X9project/X9_AI_system.git
cd X9_AI_system
```

### 2.2 准备 `.env.shared`

仓库里没有 `.env.shared`(它包含真实密钥,gitignored)。**找 Rocky 要一份开发用的 `.env.shared`**,或者按下面步骤从模板生成:

```powershell
# 从模板复制
copy .env.shared.example .env.shared
notepad .env.shared
```

**至少要填这几项**(开发环境用):

```ini
# 数据库密码 —— 跟你本地 docker-compose 启动时设的密码一致即可
X9_PG_PASSWORD=<本地任意强密码,例如 dev_local_pwd_change_me>
LOCAL_DB_URL=postgresql+psycopg://x9:<同上>@127.0.0.1:15432/x9db
X9_PG_DSN=postgresql://x9:<同上>@127.0.0.1:15432/x9db?connect_timeout=5

# AI 模型 key(开发可以用自己的或者跟 Rocky 要)
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...

# Gmail OAuth — 开发不一定需要,如果不碰外联功能可以留占位符
GMAIL_DEFAULT_CLIENT_ID=...
GMAIL_DEFAULT_CLIENT_SECRET=...
```

**不要把真实的 `.env.shared` 提交进 git。** 它已经在 `.gitignore` 里,但如果哪天有人手动 `git add -f .env.shared`,push 上去就完蛋了。

### 2.3 启动数据库(Docker)

```powershell
# 拉起 PostgreSQL 容器
cd infra\docker
docker compose up -d

# 验证容器在跑
docker ps | findstr x9-postgres
# 应该看到 "x9-postgres ... 0.0.0.0:15432->5432/tcp"
```

第一次跑会下载 PostgreSQL 16 镜像(几百 MB),需要几分钟。

### 2.4 初始化 schema

```powershell
cd ..\..\          # 回到 F:\X9_AI_system\ 等价位置(项目根)
.\infra\scripts\db_init.ps1
```

`db_init.ps1` 会创建数据库、应用所有迁移脚本。如果报错,看一下:
- `.env.shared` 里的密码和你 `docker compose` 时设置的是否一致
- 容器是否真的在跑(`docker ps`)
- 端口 15432 是否被占用(`netstat -ano | findstr 15432`)

### 2.5 安装 Python 依赖

**强烈建议用虚拟环境**,避免污染全局 Python:

```powershell
# 在项目根目录
python -m venv .venv
.\.venv\Scripts\Activate.ps1
# 如果 PowerShell 拒绝运行脚本,先以管理员开 PowerShell 跑:
# Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

# 装 core 依赖
pip install -r core\requirements.txt

# 装 desktop 依赖
pip install -r desktop\requirements.txt

# 装 scrapers 依赖
pip install -r scrapers\requirements.txt

# 装开发工具(lint/test)
pip install -r requirements-dev.txt
```

> 如果还没有 `requirements-dev.txt`,Rocky 会在第一次 CI 设置时建。临时可以:
> ```
> pip install ruff pytest pytest-asyncio black mypy
> ```

### 2.6 安装 Node 依赖

```powershell
cd web
npm ci
cd ..\web-user
npm ci
cd ..
```

`npm ci`(不是 `npm install`)会严格按 `package-lock.json` 装,保证团队所有人版本一致。

### 2.7 启动所有服务

```powershell
# 一键启动 core + desktop(已有的脚本)
.\start_all.ps1
```

浏览器会自动打开 `http://localhost:18765`(core 后台)。
desktop 后端在 `http://localhost:8000`。

前端开发模式单独跑:

```powershell
# 终端 1
cd web
npm run dev          # 通常在 http://localhost:5173

# 终端 2
cd web-user
npm run dev          # 通常在 http://localhost:5174
```

### 2.8 跑通"我能改代码"的最小验证

新人入职第二天的练习:

1. 改一行 `core/app/static/index.html` 里的标题,比如加上你的名字
2. 刷新 `http://localhost:18765`,确认改动生效
3. 改回去
4. `git diff` 看自己的改动
5. `git checkout -- core/app/static/index.html` 放弃改动

走通这一步,本地环境就算配好了。

## 3. 日常工作流(每个开发者)

### 3.1 每天开始工作

```powershell
cd D:\code\X9_AI_system

# 同步 main
git checkout main
git pull origin main

# 开新 feature 分支
git checkout -b feat/我要做的事

# 启动服务(如果昨天没在跑)
```

### 3.2 写代码 + 提交

```powershell
# 写...写...写...

# 提交前自检
git status
git diff

cd web && npm run lint    # JS/TS lint

# 提交
git add <文件>
git commit -m "feat: 加了 XYZ 功能"

# 推到远程
git push -u origin feat/我要做的事
```

### 3.3 提 PR

去 GitHub 仓库页面,会看到一个黄色横幅 "Compare & pull request",点进去,按模板填写。

### 3.4 PR 被评论了

```powershell

git add <文件>

```

### 3.5 PR 合并后清理

```powershell
git checkout main
git pull

```

## 4. 常见问题

### "端口被占用"

```powershell
# 找出占用 18765 的进程
netstat -ano | findstr 18765
# 拿到 PID,kill
taskkill /PID <PID> /F
```

### "Docker 容器跑不起来"

```powershell
docker logs x9-postgres      # 看错误日志
docker compose down          # 在 infra/docker 目录下
docker compose up -d
```

### "pip install 报 SSL 错误"

公司网络可能拦截。临时:

```powershell
pip install -r requirements.txt --trusted-host pypi.org --trusted-host files.pythonhosted.org
```

长期解决方案是配置内网镜像源。

### "我的 PR CI 挂了"

去 PR 页面点 `Details` 看具体错误。常见原因:
- Lint 不过 → 本地跑 `ruff check . --fix` 自动修
- 测试挂 → 看输出找原因
- 找不到 `.env` 里某个 key → CI 里没设那个 secret,跟 Rocky 说

### "我把 main 不小心 commit 了东西"

```powershell
# 还没 push 的话
git reset --soft HEAD~1     # 撤销最近一次 commit,改动还在
git stash                   # 把改动 stash
git checkout -b feat/我应该开的分支
git stash pop
git commit -m "..."
```

push 过了的话(还没人合)→ 立刻在 Slack 说,**不要自己 force push**,让 Rocky 处理。

## 5. 数据库迁移工作流

改 schema 不能直接在数据库里 `ALTER TABLE`,要写迁移脚本。

### 5.1 写一个迁移

```powershell
# 在 core/scripts/ 或 desktop/scripts/ 下创建
# 命名:migrate_v<版本号>_<描述>.py
# 例如:migrate_v21_add_creator_tags.py
```

迁移脚本最低限度要做到:

```python
# core/scripts/migrate_v21_add_creator_tags.py
import os
import psycopg

DSN = os.environ["X9_PG_DSN"]

def upgrade():
    with psycopg.connect(DSN) as conn:
        with conn.cursor() as cur:
            cur.execute("""
                ALTER TABLE creators
                ADD COLUMN IF NOT EXISTS tags JSONB DEFAULT '[]'::jsonb;
            """)
            cur.execute("""
                CREATE INDEX IF NOT EXISTS idx_creators_tags
                ON creators USING gin (tags);
            """)
        conn.commit()

def downgrade():
    """如何回退 —— 必须写,审查会看"""
    with psycopg.connect(DSN) as conn:
        with conn.cursor() as cur:
            cur.execute("DROP INDEX IF EXISTS idx_creators_tags;")
            cur.execute("ALTER TABLE creators DROP COLUMN IF EXISTS tags;")
        conn.commit()

if __name__ == "__main__":
    import sys
    if "--down" in sys.argv:
        downgrade()
    else:
        upgrade()
    print("OK")
```

### 5.2 跑本地迁移

```powershell
python core\scripts\migrate_v21_add_creator_tags.py
```

### 5.3 提 PR

迁移脚本的 PR **必须 Rocky review**(在 CODEOWNERS 里强制了)。Review 时会重点看:
- 有没有 `IF NOT EXISTS`(让脚本可重复跑)
- 有没有 downgrade
- 有没有可能锁大表
- 有没有 backfill(给老数据填新字段)

### 5.4 生产应用迁移

PR 合到 main 之后,Rocky 在生产克隆里:

```powershell
cd F:\X9_AI_system
git pull origin main
python core\scripts\migrate_v21_add_creator_tags.py
.\start_all.ps1   # 重启服务
```

详见 04 文档的部署章节。

## 6. 维护者(Rocky)的特殊配置 — 同机 dev/prod 分离

你"自己开发"+ "这台机器是公司服务器"的双重身份必须在物理上分开,否则一次手抖就影响所有人。

### 6.1 目录布局

```
F:\X9_AI_system\           ← 生产克隆(team 用的"上线版本")
                             分支:只在 main 上
                             端口:postgres 15432, core 18765, desktop 8000
                             数据库:x9-postgres 容器,数据卷 x9_pg_data

F:\X9_dev\X9_AI_system\    ← 你的开发克隆
                             分支:feat/xxx
                             端口:postgres 25432, core 28765, desktop 18000
                             数据库:x9-postgres-dev 容器,独立数据卷 x9_pg_data_dev
```

### 6.2 创建 dev 克隆

```powershell
mkdir F:\X9_dev
cd F:\X9_dev
git clone git@github.com:X9X9project/X9_AI_system.git
cd X9_AI_system
```

### 6.3 dev 专用的 `.env.shared`

复制 `.env.shared.example` 到 `.env.shared`,**改端口和数据库密码**:

```ini
# 注意端口都加了 10000 以避开生产
X9_PG_PORT=25432
X9_PG_DSN=postgresql://x9:dev_only_password@127.0.0.1:25432/x9db?connect_timeout=5
X9_PG_PASSWORD=dev_only_password
LOCAL_DB_URL=postgresql+psycopg://x9:dev_only_password@127.0.0.1:25432/x9db

X9_CORE_PORT=28765
X9_DESKTOP_PORT=18000
```

### 6.4 dev 专用的 docker-compose

模板已经在 `docs/team/templates/infra/docker-compose.dev.yml`,复制到:

```powershell
copy docs\team\templates\infra\docker-compose.dev.yml infra\docker\docker-compose.dev.yml
```

启动 dev 数据库(注意是用 dev 的 compose 文件):

```powershell
cd F:\X9_dev\X9_AI_system\infra\docker
docker compose -f docker-compose.dev.yml up -d
```

这样会启动一个**完全独立**的容器 `x9-postgres-dev`,端口 25432,跟生产容器 `x9-postgres` 互不干扰。

### 6.5 dev 数据初始化

```powershell
# 在 dev 克隆里
cd F:\X9_dev\X9_AI_system
.\infra\scripts\db_init.ps1   # 会读 .env.shared 里的连接信息,连到 dev DB

# 把生产数据 dump 过来作为开发数据(可选)
docker exec x9-postgres pg_dump -U x9 x9db > F:\X9_dev\prod_snapshot.sql
docker exec -i x9-postgres-dev psql -U x9 x9db < F:\X9_dev\prod_snapshot.sql
```

### 6.6 日常工作

**写代码** → 在 `F:\X9_dev\X9_AI_system\` 里
**部署** → 在 `F:\X9_AI_system\` 里 `git pull origin main` + 重启

```powershell
# 部署脚本(放到 F:\X9_AI_system\infra\scripts\deploy.ps1)
$ErrorActionPreference = 'Stop'
cd F:\X9_AI_system
git fetch origin main
$current = git rev-parse HEAD
$target  = git rev-parse origin/main
if ($current -eq $target) { Write-Host "Already up to date"; exit }

Write-Host "Backing up database..."
docker exec x9-postgres pg_dump -U x9 x9db > "F:\backup\pre_deploy_$(Get-Date -Format yyyyMMdd_HHmmss).sql"

Write-Host "Pulling..."
git pull origin main

Write-Host "Applying migrations (review output)..."
# 这里需要根据本次 commit 找出新的 migration 脚本,人工执行
# 自动检测的话可以维护一个 migrations 目录,见 04 文档

Write-Host "Restarting services..."
# stop_all 然后 start_all
.\start_all.ps1

Write-Host "Deploy done."
```
