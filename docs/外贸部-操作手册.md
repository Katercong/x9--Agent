# 外贸部系统 — 操作手册（极简）

> 命令整段复制到 PowerShell 回车即可。本机验证用 `http://localhost:8000`；上线靠 `git push` 后同事服务器部署。

## 1. 启动系统
```powershell
powershell -ExecutionPolicy Bypass -File "F:\Claude_Project\X9_AI_system_fresh\start_all.ps1"
```
看到 `OK Desktop` 即成功。浏览器开 `http://localhost:8000/login`。

## 2. 登录账号（用本系统已有账号）
用你本地数据库里的账号登录即可，按角色/部门看到对应界面：

| 账号类型 | 登录后 |
|----------|--------|
| 外贸部·部门管理员 | 外贸部管理端（招聘+社媒） |
| 外贸部·成员 | 外贸部门户（含「下载插件」） |
| 跨境部账号 | 跨境部（原 TikTok，不变） |
| 超级管理员 | 全局 |

> 账号密码以各自部署的实际数据为准，不在文档中固化。本地若需测试账号，可自行用超管在「用户与权限」里创建。

## 3. 改了前端后重新部署
```powershell
cd F:\Claude_Project\X9_AI_system_fresh\web;      npm run build:root;   npm run deploy:root
cd F:\Claude_Project\X9_AI_system_fresh\web-user; npm run build:deploy; npm run deploy
```
完成后浏览器 `Ctrl+Shift+R` 硬刷新。（首次或换依赖才需先 `npm install`；后端不用重启。）

## 4. 改了后端后
重启 desktop（重跑第 1 步）才生效。新表会自动建，无需迁移。

## 5. 下载并安装外贸插件
1. `ftuser` 登录门户 → 点「下载插件」→ 解压。
2. Chrome → `chrome://extensions` → 开发者模式 → 加载已解压的扩展程序 → 选解压文件夹。
3. **逐页采集（即开即用）**：打开并登录 51job/智联/大泉州/小红书/抖音页面，插件自动采当前页 → 回后台「公司客户线索 / 社媒线索」查看。

## 6. 批量采集（关键词自动翻页，需先装 helper）
```powershell
powershell -ExecutionPolicy Bypass -File "F:\Claude_Project\X9_AI_system_fresh\desktop\foreign-trade-helper\install_ft_helper.ps1"
```
装完后**必须杀掉旧 helper 进程**（它常驻、不会自动重读新配置，否则一直用旧设置导致退出码 1）：
```powershell
Get-NetTCPConnection -State Listen -LocalPort 8765 -ErrorAction SilentlyContinue | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```
再重启 Chrome → 插件侧边栏「招聘网站采集 → 批量采集 → 开始采集」（helper 会被自动重新唤起并读新配置）。
（需本机已装 Python 3.11+ 和 Chrome；`[3/4]` 下载 Chromium 慢几分钟正常。智联/人才端需在弹出的受控窗口登录。）

## 7. 上线给同事部署
`git push` 后，同事在其服务器：
```powershell
git pull
cd web;      npm install; npm run build:root;   npm run deploy:root
cd ..\web-user; npm install; npm run build:deploy; npm run deploy
```
再重启 desktop。数据库不动（只增不删）。

## 8. 出问题回退插件/helper 改动
```powershell
powershell -ExecutionPolicy Bypass -File "F:\Claude_Project\_X9_rollback\20260601_082920_before_helper\RESTORE_回退.ps1"
```

---

**常见问题**
- 脚本报"未对脚本进行数字签名"：用上面带 `-ExecutionPolicy Bypass` 的整行命令，别直接敲 `.\xxx.ps1`。
- 端口已占用 / already running：正常，不用管。
- 侧边栏「helper 未连接」：批量采集才需要 helper（见第 6 步）；逐页采集不需要。
- 批量采集**退出码 1 / helper root 还指向旧目录**：旧 helper 进程没刷新。杀掉它即可：`Get-NetTCPConnection -State Listen -LocalPort 8765 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }`，再点「开始采集」。
- 数据库没起：确认 Docker Desktop 已开，重跑第 1 步。
